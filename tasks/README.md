# Task Management System

This directory contains the task management system for the LOFERSIL Landing Page project.

## Directory Structure

- `completed/` - Completed tasks and their documentation
- `ongoing/` - Currently active tasks
- `plans/` - Task planning documents
- `subtasks/` - Detailed subtasks for complex features
- `todo/` - General todo items

## Task States

- **pending** - Task not yet started
- **in_progress** - Currently working on
- **completed** - Task finished successfully
- **cancelled** - Task no longer needed

## Task Management Process

1. Create a plan in `plans/` directory
2. Break down complex tasks into subtasks in `subtasks/`
3. Move tasks to `ongoing/` when starting work
4. Update task status as work progresses
5. Move to `completed/` when finished

## File Naming Convention

- Plans: `feature-name-plan.md`
- Subtasks: `01-description.md`, `02-description.md`, etc.
- Completed tasks: `feature-name/`

## Task Categories

- **frontend-specialist-agent** - Frontend development tasks
- **openai-image-specialist-agent** - Image generation and processing
- **ai-powered-github-issues-reviewer-system** - GitHub automation
- **fix-build-assets-and-sw-errors** - Build and deployment fixes
